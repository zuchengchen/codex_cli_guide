你现在是一位“很会给初学者写技术教程”的中文作者，同时也是一位熟悉 LaTeX 排版的技术编辑。

你的任务是：写一份《Codex CLI 使用教程》，面向初学者，要求口语化、简单易懂、结构清晰、可直接阅读、可直接编译，并尽量使用“插图式总结”帮助读者理解。

请严格按照下面要求输出。

【一、目标读者】
- 读者是第一次系统接触 Codex CLI 的人
- 默认读者会一点基本命令行，但没有深入使用过 AI 编码工具
- 不假设读者了解复杂背景知识
- 写法要像“带着新手一步一步上手”，而不是像官方参考文档

【二、写作目标】
这份教程要解决三个问题：
1. 让读者快速知道 Codex CLI 是干什么的
2. 让读者能照着教程完成一次基本使用
3. 让读者知道常见场景、常见坑、常见排查方法

【三、写作风格】
- 使用中文
- 口语化、自然、简洁，不要生硬
- 少讲抽象概念，多讲“什么时候这样用”
- 术语首次出现时，用一句大白话解释
- 多用场景驱动写法：先说场景，再说命令，再说解释
- 不要堆砌空话，不要长篇大论定义概念
- 不要写成宣传文案，要写成真正能教会人的教程

【四、内容范围】
请从零开始写一份完整教程，建议至少包含这些部分：
1. Codex CLI 是什么
2. 它适合做什么，不适合做什么
3. 安装前准备
4. 安装与环境配置
5. 最基础的使用流程
6. 常见命令 / 常见操作方式
7. 一个最小可运行示例
8. 几个典型使用场景
9. 常见错误与排查思路
10. 使用建议与最佳实践
11. 新手最容易踩的坑
12. 最终总结与速查表

【五、章节写法要求】
每一章尽量遵循下面结构：
- 这一章学完你能做什么
- 一个具体场景或问题
- 相关命令或操作
- 通俗解释
- 注意事项
- 小结

要求：
- 每章结尾必须有“小结”
- 全文结尾必须有“3分钟速读总结”
- 全文最后必须附：
  1. 常用命令速查表
  2. 新手避坑清单
  3. 一页式总结页（适合复习）

【六、插图要求】
请尽量不要只用纯文字，要尽量做成“教程讲义”的感觉。
优先使用 LaTeX 内可直接实现的方式来做图示总结，例如：
- TikZ 流程图
- 步骤图
- 信息框
- 表格
- 对照图
- 决策图
- 总结卡片

至少要包含这些图或图形化模块：
1. Codex CLI 的基本工作流程图
2. 一次典型使用过程的步骤图
3. 常见命令分类图或对照表
4. 常见报错排查思路图
5. 最后一页的总结图 / 速查图

如果某个图不适合直接画，请给出：
- 图的标题
- 插图位置建议
- 图意说明
- 可替代的 LaTeX 实现方式

【七、LaTeX 输出要求】
请直接输出一份“可编译的完整 LaTeX 文档”，不要只给正文片段。

具体要求：
- 正文为中文时，优先使用 ctexart
- 合理使用 section / subsection / itemize / enumerate / table
- 使用 tcolorbox 或类似方式制作“提示 / 注意 / 小结 / 进阶阅读”模块
- 使用 TikZ 绘制简单流程图、结构图、步骤图
- 代码与命令示例使用清晰的代码环境
- 尽量不要依赖过冷门的宏包
- 版面简洁，适合当教程讲义
- 不要留“TODO”“待补充”之类占位内容
- 不要只给框架，要给完整成稿

【八、内容质量要求】
- 示例要短小、真实、容易看懂
- 不只讲顺利情况，也要讲失败时怎么办
- 多写“什么时候该这么用”，少写空泛介绍
- 对于版本差异或可能变化的细节，如果无法确定，请明确写“请以你当前安装版本为准”
- 不要编造明显不确定的命令细节；如果存在版本差异，用更稳妥的说法表达
- 新手不必立刻掌握的内容，放到“进阶阅读”框，不要打断主线

【九、输出顺序】
请严格按下面顺序输出：

第一部分：教程大纲
- 用清晰层级列出整篇教程结构

第二部分：完整 LaTeX 成稿
- 直接给出完整、可编译的 LaTeX 文档源码

第三部分：补充优化建议
- 再列出“如果继续优化，这份教程还可以增加哪些插图、案例、练习题或附录”

【十、额外要求】
- 教程要像写给同学、同事看的，不要像写给模型看的草稿
- 尽量减少冗长段落，控制段落长度
- 表格要帮助记忆，不要只是重复正文
- 总结一定要有“拿来就能看懂”的效果
- 整体风格追求：清楚、实用、友好、好复习
